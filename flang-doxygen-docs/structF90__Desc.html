Detailed Description

Fortran descriptor header.

The fields minus F90_DescDim below should remain consistent with the object_desc and proc_desc structures of type.h in terms of length and type. These fields are also mirrored in the Fortran Front-end's rte.h header file.

The tag field is used to identify the type of the descriptor. This is typically __DESC, which identifies the descriptor as a regular array section descriptor. If the tag is a basic type, such as an __INT4, \ref__REAL4, etc. then it is a 1 word pseudo descriptor. The pseudo descriptor is used as a place holder when we want to pass a scalar into a runtime routine that also requires a descriptor argument. When tag is __POLY, then we have an object_desc (see its definition in type.h). When tag is __PROCPTR, we have a proc_desc (see its definition in type.h).

The rank field equals the total number of dimensions of the associated array. If rank is 0, then this descriptor may be associated with a derived type object, a pointer to a derived type object, or an allocatable scalar.

The kind field holds the base type of the associated array. It is one of the basic types defined in _DIST_TYPE.

The flags field holds various descriptor flags defined above. Most of the flags defined above, denoted as reserved, are not used for Fortran. The flags that are typically used for Fortran are __ASSUMED_SIZE and __ASSUMED_SHAPE.

The len field holds the byte length of the associated array's base type (see also kind field).

The lsize field holds the total number of elements in the associated array section.

In distributed memory languages, such as HPF, gsize represents the total number of elements that are distributed across multiple processors. In Fortran, the lsize and gsize fields are usually the same, however this is a case in the reshape intrinsic where gsize != lsize. There may just be an incidental difference during the execution of reshape. There may also be others in the Fortran runtime where gsize != lsize, however, they too may just be incidental differences. Therefore, use lsize instead of gsize when querying the total number of elements in a Fortran array.

The lbase field is the index offset section adjustment. It is used in the mapping of an array section to its original array.
See the __DIST_SET_SECTIONXX and __DIST_SET_SECTIONX macros below
for examples of how to compute this field. See the __fort_print_local() runtime routine in dbug.c for an example of how to use this field.

The gbase field historically was used in distributed memory languages like HPF. Therefore, gbase is usually 0 and may always be 0 in the
Fortran runtime (needs more investigation to confirm if it's always 0 in this case).

When set, the dist_desc field holds a pointer to the type descriptor of the associated object (see also the TYPE_DESC definition in type.h).

The dim fields hold up to number of F90_DescDim structures. It's also possible that dim is empty when this descriptor is associated with a derived type, a pointer to a derived type, or an allocatable scalar.

The number in paranthesis for each field below corresponds with the subscript index in the descriptor. The first 9 values are denoted. The first index is 1 because we assume Fortran style arrays when we reference these fields in the Fortran front-end. When generating assembly, the first index will be 0. After the gbase field, the subscript value depends on three conditions: Whether the target's pointers are 64-bit, whether the target's native integers are 64-bit, and whether large arrays are enabled. See the DESC_HDR_LEN macro in the Fortran front-end's rte.h file for more information. The first 9 subscript values are also mirrored in the following macros in rte.h: DESC_HDR_TAG, DESC_HDR_RANK, DESC_HDR_KIND, DESC_HDR_BYTE_LEN, DESC_HDR_FLAGS, DESC_HDR_LSIZE, DESC_HDR_GSIZE, DESC_HDR_LBASE, and DESC_HDR_GBASE.
