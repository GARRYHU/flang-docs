8/******************************************************************************
9 * *
10 * Background: *
11 * The POWERPC ABI does not provide for tail calls. Thus, the math dispatch *
12 * table processing incurs overhead with the saving and restoration of GPR 2 *
13 * that can severely affect application performance. For POWERPC, we use an *
14 * optimized assembly dispatch set of routines that make tail calls to all of *
15 * the routines defined in the math dispatch configuration files but do not *
16 * saveand /restore GPR 2. *
17 * *
18 * !!!!!!!!!!!!!!!!!!!!!!!!!!!!!!! WARNING !!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!! *
19 * *
20 * If any entry (routine <FUNC>) in any of the dispatch tables is not present *
21 * in i.e. not satisfied by, libpgmath, in order to properly preserve/restore*
22 * GRP 2 when calling routine <FUNC>, the actual function must first be *
23 * encapsulated in a routine present in libpgmath. *
24 * *
25 * No doubt there are pathological cases that will show this engineering *
26 * choice to be wrong, but current performance testing shows otherwise. *
27 * *
28 *****************************************************************************/
