114 * WARNING -- changing IVSN means that old .mod files can't be read.

117 *
118 * HISTORY:
119 * <= 24 - 10.6 & before
120 * 25 - add DT_DEFERCHAR and DT_DEFERNCHAR, 32 more SYM flags, and
121 * another SYM field --- briefly used in DEV only, NOT released,
122 * compatibility FAILS
123 * 26 - add DT_DEFERCHAR and DT_DEFERNCHAR, 32 more SYM flags, and
124 * 3 more SYM fields. Prefix the set of new flags & fields with
125 * ' A' to help with compatiblity with IVSN 24
126 * 27 - add platform(x86 and x86-64 after version number. This is to
127 * to prevent 32-bit module being used by 64-bit compiler and
128 * vice versa.
129 * 28 - 32 more SYM flags, and 3 more SYM fields. Prefix the set of
130 * `new flags & fields with
131 * ' B' to help with compatiblity with IVSN 24
132 * 29 - Embed descriptor for scalar "non-polymorphic" pointer members in
133 * the derived type object. We do this for consistency with other
134 * members including polymorphic scalars.
135 * 30 - Add more 1 AST field, w18; used to record the 'end label' of
136 * various MP ASTs such as PARALLEL/PDO/TASK/SECTIONS.
137 * 31 - Add MP_ATOMICxxx for atomic operations
138 * 32 - add compiler own module files flag into platform flag.
139 * It is set if it is compiler module file.
140 * 33 - Add MP_TASKLOOP[REG] for taskloop
141 * 34 - half precision and half-complex datatypes
142 * 35 - Add flag for null subc in typedef initializer
143 */

145/*
146 * Interface file format:
147 *
148 * line 1 - Vversionnumber module-name
149 * line 2 - source-file-name-len source-file-name Sstb.firstosym
150 * line 3 - time-date-stamp
151 *
152 * for module files:
153 * zero or more 'use' lines:
154 * - use module-name [scope]
155 * where 'module-name' is the textual module name, and 'scope' is an integer
156 * greater than zero if the module is used at the start of the module
157 * and should be transitively used in the using program.
158 *
159 * Remaining lines - records begin with a 'letter':'
160 *
161 * - - comment line
162 * A - ast definition line
163 * B - based symbol definition line
164 * C - reserved for constructors
165 * D - data dtype definition line
166 * E - equivalence line
167 * F - formal arguments
168 * G - word of an align descriptor
169 * H - shadow information for an array
170 * I - data initialization
171 * L - storage overLap list
172 * M - mangled derived symbol descriptor
173 * N - namelist members
174 * O - overloaded (generic) descriptor
175 * P - module predeclared (e.g. ST_HL, ST_HLL) symbol becomes predeclared
176 * Q - module procedure descriptor
177 * R - Rename line for variables in used modules
178 * S - symbol definition line
179 * T - shape descriptor
180 * U - element of a distribute record
181 * V - STDs
182 * W - ast arg table
183 * X - asd (subscript) table
184 * Y - ast list items
185 * Z - end of file
186 *
187 * ******
188 * For static initialization files, there is another section,
189 * following the Z line, containing initialization info:
190 * A - ast pointer to value reference in data statement
191 * D - DO start in data statement
192 * E - DO end in data statement
193 * I - variable initialization in type statement
194 * L - literal integer
195 * J - start of data statement
196 * O - implied do initializer
197 * S - typedef initializer (subc used)
198 * R - aRray initializer (subc used)
199 * V - Variable being initialized in data statement
200 * W - subtype pointer to variable being initialized
201 * X - eXpression initializer
202 */
