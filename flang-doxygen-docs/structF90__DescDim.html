Detailed Description

Fortran descriptor dimension info.

Each F90_Desc structure below has up to MAXDIMS number of F90_DescDim structures that correspond to each dimension of an array. Each F90_DescDim has 6 fields: lbound, extent, sstride, soffset, lstride, and ubound.

The lbound field is the lowerbound of the dimension.

The extent field is the extent of the dimension (e.g., extent = max((ubound - lbound) + 1), 0).

Fields sstride (i.e., section index stride on array) and soffset (i.e., section offset onto array) are not needed in the Fortran runtime. They were needed in languages like HPF. For Fortran, their corresponding macros, F90_DPTR_SSTRIDE_G and F90_DPTR_SOFFSET_G are set to constants (see below). However, we still need to preserve the space in F90_DescDim for backward compatibility.

The field lstride is the "section index multiplier" for the dimension. It is used in the mapping of an array section dimension to its original array. See the __fort_finish_descriptor() runtime routine in dist.c for an example of how to compute this field. See the print_loop() runtime routine in dbug.c for an example of how to use this field.

The ubound field is the upperbound of the dimension.
