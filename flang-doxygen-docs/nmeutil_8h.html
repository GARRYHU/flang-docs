◆ add_rpct_nme()

int add_rpct_nme ( int  orig_nme,
int  rpct_loop 
)

...

This creates and returns a new NME, 'rpct_nme', whose fields are the same as those of 'orig_nme' except that it has NME_RPCT_LOOP( rpct_nme ) == rpct_loop (> 0), whereas NME_RPCT_LOOP( orig_nme ) == 0. 'rpct_nme' will replace all occurrences of 'orig_nme' within the loop denoted by 'rpct_loop', whereas 'orig_nme' will continue to be used elsewhere. This loop is guarded by runtime pointer conflict tests ('RPCT's) which prove at runtime that the 'rpct_nme' references don't conflict with various other references in the loop that they could potentially conflict with, as indicated in the 'RPCT table', and 'conflict()' will use this information to return NOCONFLICT for pairs of such references.
