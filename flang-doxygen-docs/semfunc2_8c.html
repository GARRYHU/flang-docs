◆ chkarg()

int chkarg ( SST *  stkptr,
int *  dtype 
)

Performing checking on an argument to determine if it needs to be "protected".

Parameters
stkptra stack entry representing a subprogram argument
dtypeused to pass out data type of argument
Returns
sptr for alternate return label

When an argument needs to be protected, its value may be have to be stored in a temp and then the temp's address becomes the actual argument. This occurs when a parenthesized expression is an actual argument. NOTE that the logic in chkarg relies on an argument being <expression>; a flag (SST_PARENG/P) in the semantic stack is used and is only defined for <expression>; the flag is cleared when <expression> ::= ... occurs and is set when <primary> ::= ( <expression> ) occurs.

Process information for deferred interface argument checking in in the compat_arg_lists() function below.

If the performChk argument is false, then we save the information (defer the check). If performChk argument is true, then we perform the argument checking. Note: If performChk is true, then the other arguments are ignored.

Parameters

actualis the symbol table pointer of the actual argument.
flagsare comparison flags that enable/disable certain checks
linenois the source line number for the deferred check
performChkis false to defer checks and true to perform the checks.
