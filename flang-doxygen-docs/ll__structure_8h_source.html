578/**
579 \brief LLVM Metadata
580
581 Metadata nodes are constant manifestly typed tuples of references to other
582 metadata nodes, metadata strings, or LLVM constants. The values that can be
583 stored in a metadata node are represented as an array of LL_MDRef references.
584
585 Metadata nodes and strings are normally uniqued to save space, but it is
586 possible to create distinct metadata nodes that don't participate in
587 uniquing.
588
589 Metadata can appear as annotations on instructions, or as module-global
590 named metadata.
591
592 LL_MDRef is a compact manifestly typed reference to the kind of data that can
593 be stored in a metadata node. An LL_MDRef can refer to:
594
595 \li A numbered metadata node, or \c NULL.
596 \li A metadata string.
597 \li A constant LLVM value.
598
599 LL_MDRef is passed around by value. It should be treated as an opaque type.
600 It cannot be transferred between different LL_Module instances.
601 */
602typedef unsigned LL_MDRef;
603
604#define LL_MDREF_kind(md) ((md) & ((1 << 3) - 1))
605#define LL_MDREF_value(md) ((md) >> 3)
606#define LL_MDREF_ctor(k, v) (((v) << 3) | LL_MDREF_kind(k))
607#define LL_MDREF_INITIALIZER(key, val) LL_MDREF_ctor(key, val)
608#define LL_MDREF_IS_NULL(md) ((md) == 0)
